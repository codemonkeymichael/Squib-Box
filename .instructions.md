# Squib-Box DMX Receiver Project

## Overview
DMX receiver system using Raspberry Pi Pico + MAX485 RS485 transceiver, controlled via QLC+ (Q Light Controller Plus 4.13.1) running on Enttec DMX USB Pro.

## Hardware Setup
- **Pico UART1**: 250k baud, 8-bit, no parity, 2 stop bits (standard DMX512)
- **MAX485 module**: 
  - RO (Receiver Output) → Pico GPIO 5 (UART1 RX)
  - DE (Driver Enable) → GND (receive-only mode)
  - RE (Receiver Enable) → GND (receive-only mode)
  - Power: 3.3V from Pico
  - Ground: Common with Pico

## GPIO Assignments (Triggers)
- **GPIO 2**: Channel 1 (falling edge / turn OFF)
- **GPIO 3-4, 6-18**: Channels 2-16 (rising edge / turn ON)
- **GPIO 5**: Reserved (UART RX - DO NOT USE)

## Key Technical Findings
### Frame Alignment Issue
- Data arrives with **16-byte offset** due to UART buffering/break detection timing
- Channels appear at data[17:22] in raw read, but detected start code (0x00) indicates channel positions in frame
- Solution: Detect DMX break (gap >500µs) and assume data[1:17] are channels 1-16

### Debouncing Implementation
- **3-frame debouncing** for theater-grade reliability (~70-75ms delay, imperceptible)
- Validates start code (0x00 only)
- Only reports readings when all 3 consecutive frames match
- Edge detection: fires GPIO pulse on state transition

### Channel Behavior
- **Channel 1 (GPIO 2)**: Falling edge (ON→OFF transition)
  - Keep high (>100) during show
  - Turn OFF to trigger cue
  - GPIO fires when it goes OFF
  
- **Channels 2-16 (GPIO 3-18)**: Rising edge (OFF→ON transition)
  - Keep low (≤100) during show
  - Turn ON to trigger each effect
  - GPIO fires when it goes ON

## QLC+ Configuration Notes
- Enttec DMX output mode required (not Art-Net/sACN)
- Channels set to 255 (full) for ON state, 0 for OFF
- Operator assigns cues to turn specific DMX channels on/off

## Current Script Behavior
- Continuous DMX receive loop
- 3-frame debounce validation
- Individual edge detection per channel
- 250ms GPIO pulse width per trigger
- Prints first 5 channels for monitoring

## Relay Wiring
Each GPIO output needs a relay circuit (typical setup):
```
GPIO → 470Ω resistor → Transistor base (or relay coil directly if low-power)
Relay coil → +12V supply (or appropriate voltage)
Relay coil → Flyback diode (1N4007) to GND
Relay contacts → Load (lights, motors, etc.)
```

## Troubleshooting
- If channels read zero: Check MAX485 wiring (A/B polarity, ground continuity)
- If triggers miss: Increase frame buffer to 5 frames (adds ~120ms delay)
- If relays chatter: Increase GPIO pulse width (currently 250ms)
